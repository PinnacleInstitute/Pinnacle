Email Messages

Email Messages are defined, scheduled and tracked for each email marketing activity.  Each email message specifies a name, target audience, subject, message, send date, status, and recurrence schedule.  The email scheduler reviews all defined email messages nightly to determine which ones need to be sent.  Nonrecurring email messages are marked complete after being sent.  Recurring email messages are marked complete after their end date is past.

An email message can be formatted with a WYSIWYG HTML Editor and can include images, tables, links, etc.  The message can also include field references for the specific member information.  The reference fields are identified with curly brackets and include {firstname}, {lastname}, {email}, {data1}, {data2}, {data3}, {data4}, and {data5}.  Sections of text can also be included or excluded based on the values of data1, data2,  data3, data4 and data5.  See "Included / Excluded Text" below for more details.

An email message can be sent multiple times by entering a recurring schedule.  The email scheduler will send all messages nightly that have a status of ready and a blank send date or a send date earlier or equal to the current date. The email message can be repeated by entering a number followed by a single letter (d,w,m) for a (day,week,month).  For example, 7d will send the message once every seven days, and is the same as 1w.  An end date can be specified to stop sending the email message.  If no end date is specified the email message will be sent indefinitely according to the recurring schedule.

The Email message can be sent to a test email address to confirm the composition of the email message.  The email test can also include a test firstname, lastname, data1, data2, data3, data4, data5.

Each email message tracks the total number of sent emails and the total number of mailings according to the recurring schedule.

 

Included / Excluded Text

Sections of text in the body of the email message can be included or excluded based on the values of data1, data2, data3, data4 or data5.  The section of text is encompassed within two matching tags that define the criteria for inclusion or exclusion.  The following options are supported.

{data1:+}                        include if there is any value

{data1:-}                        include if there is no value

{data1:<xxx}                        include if the value is less than xxx

{data1:>xxx}                        include if the value is greater than xxx

{data1:xxx}                        include if the value is equal to xxx

{data1:else}                        include if no other previous condition was satisfied

 

Examples

{data1:+} this text will be displayed if data1 has any value {data1:+}

{data2:-} this text will be displayed if data2 has no value {data2:-}

{data3:<5} this text will be displayed if data3 is less than 5 {data3:<5}

{data1:bob} this text will be displayed if data1 equals bob {data1:bob}
