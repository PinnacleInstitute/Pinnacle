Get Started with Macros
Step 1: Open a Database

The first thing you must do is open one of your databases. Click File and scroll to Open. Find the database you wish to use and click Open.
Step 2: Starting from Insert

Click Insert and scroll to Macros.
Step 3: Looking at the Macro Window

The Macro window opens. The arrow designates the active line. There is an area for Comment, Action, and Action Arguments. You can also maximize, minimize or close the Macro window with the usual Windows icons.
Step 4: What Is a Comment?

A Comment is something you write to assist in identifying something and how it is used. A Comment is optional.
Step 5: What Is an Action?

Actions are the steps you want Access to perform when you run your Macro.
Step 6: What Are Arguments?

Arguments allow you to define additional information for your Macro.

Build a Macro Group
Step 1: Create an Empty Macro

Start with an empty macro. Click Macros in the Objects list. Click New or Click Insert and scroll to Macros. The Macro window opens.
Step 2: Select Macro Names

Click the Macro Names icon.
Step 3: Naming the Macro Group

The macro windows have changed. They are Macro Name, Action, and Comment. Enter a name for your macro group in the first line under Macro Name.
Step 4: Adding Actions

Click the first line under Actions. Enter the desired action and set the arguments. Continue to add as many as you would like in this macro group.
Step 5: Save Your Macro Group

Click the save icon to save your macro group.

Save and Open a Macro
Step 1: Using the Save Icon

Click the Save icon.
Step 2: Save by Closing the Window

Click the X. A dialog box appears prompting you to save the Macro. Click Yes to save.
Step 3: Name the Macro

A save as box appears. Enter a name for your macro and click OK.
Step 4: Open Your Macro

In Objects, click Macros, highlight your new macro and click Design.

Work with Macros
Step 1: Run Your Macro

In Objects, click Macros and highlight your new macro. Right-click and select Run.
Step 2: Move an Action

Click the selector button (small black arrow) on the left of your action and click. Hold your mouse button down while you drag and drop it to the new location.
Step 3: Copy an Action

Highlight the Action. Right-click and scroll to Copy the action
Step 4: Paste an Action

Choose a new line and highlight it. Right-click and scroll to Paste. You will probably want to rename this macro.
Step 5: How to Delete an Action

Click the selector button (small black arrow) on the left of your action and click. Press the Delete key on your keyboard or right click and select Cut.

Create a Macro from Objects
Step 1: Select Macros

Click Macros in the Objects list.
Step 2: How to Start a New Macro

Click New. The Macro window opens.
Step 3: Assigning an Action

Click the first cell (line) in the Action column. Enter an action or click the down arrow to select an action.
Step 4: Add a Comment

Click the first cell (line) in the Comment column. Enter your Comment.
Step 5: Add an Argument

The action you choose (Rename) supplies the argument boxes. Key a New Name and click the Object Type box. Either enter your own description or click the down arrow for choices. Click the down arrow to choose the Old Name.

Use the Input Mask Wizard
Step 1: Use Design View

If you are working with a table in Datasheet view and find data such as phone numbers or Social Security numbers hard to work with because there are no dashes between groups of numbers or parentheses around area codes. You can format the numbers to help them make more sense. To change the Format field property, click the View button to switch to Design view.
Step 2: Select the Field

Click the Field Name column of the field you want to edit. Notice that the current field indicator displays to the left of the current field name, as shown in this screen. The property settings for the selected field are displayed in the lower part of the window.
Step 3: Use the Input Mask Wizard

Click in the Input Mask field property to activate the property button. Click the property button to start the Wizard. Save the table as directed by Access, and the first screen of the Input Mask Wizard displays.
Step 4: Choose a Format

Choose the desired Input Mask from the list of masks in the left column. The formatting for the input mask is shown in the right column. If you want to see how a selected mask works, first select it and then type sample data in the Try It field. Access uses the Input Mask to format the sample data. Once you have selected the desired mask, follow through the rest of the wizard by clicking the Next button, or click the Finish button at any point to use the selected mask.
Step 5: Switch to Datasheet View

From Design view click the View button. The button appears with a drop-down arrow to display the text name for the button as well as the icon. You are prompted by Access to save the table design. If you choose Yes, the changes you made will be saved and displayed in Datasheet view. If you choose No, the changes you made will not be saved, and the design in Datasheet view will not be changed.

Set a Validation Rule
Step 1: Use Design View

If you are working with a table in Datasheet view and find that you want to make sure that data that is entered into a field meets a condition, you can set a Validation Rule. For example, you might want Access to check that there are no dates before 1900. If the data being entered into a field doesn't meet the condition set in the Validation rule. Access will not allow it to be entered and will display an error message. To set a Validation Rule, click the View button to switch to Design view.
Step 2: Select the Field

Click in the field name in the Field Name column of the field you want to edit. Notice that the current field indicator displays to the left of the current field name, as shown in this screen. The property settings for the selected field are displayed in the lower part of the window.
Step 3: Type the Validation Rule

Click in the Validation Rule property field and type the rule or condition that the data in the field must meet. The condition you set up is like a simple mathematical equation. For example, if the field should contain data less than 100, you would type <100. If the field should have data greater than 100, you would type >100. You must always enclose text values in quotes and date values in the pound sign. For example, for dates on or after January 1, 1950, you would type >=#1/1/50#.
Step 4: Switch to Datasheet View

From Design view click the View button. The button appears with a drop-down arrow to display the text name for the button as well as the icon. You are prompted by Access to save the table design. If you choose Yes, the changes you made will be saved and displayed in Datasheet view. If you choose No, the changes you made will not be saved, and the design in Datasheet view will not be changed.

Set an Index
Step 1: Use Design View

If you are working with a table in Datasheet view you might find that you want to set an index to search for data more efficiently. An index is an internal table that contains only two fields: the value of the field being indexed and the location of each record in the table. Access can use the index to find matches to your requests faster by searching the indexed table first. To set an index, click the View button to switch to Design view.
Step 2: Select the Field

Click in the Field Name column of the field you want to edit. Notice that the current field indicator displays to the left of the current field. The property settings for the selected field are displayed in the lower part of the window.
Step 3: Set the Index

Click in the Indexed property field and then click the drop-down arrow. Access offers two Yes options. Yes (Duplicates OK) is used when the field may contain the same value in different records, such as the city or state field. The Yes (No Duplicates) is used when each field must contain unique data in the field, such as Social Security numbers. The primary key is always defined with this option.
Step 4: Switch to Datasheet View

From Design view click the View button. The button appears with a drop-down arrow to display the text name for the button as well as the icon. You are prompted by Access to save the table design. If you choose Yes, the changes you made will be saved and displayed in Datasheet view. If you choose No, the changes you made will not be saved, and the design in Datasheet view will not be changed.

Set a Primary Key
Step 1: Use Design View

If you are working with a table in Datasheet view, you might find that you want to set a Primary Key. Every table in an Access database should have a primary key. The field that is designated as the primary key has unique contents for each record in the table, thus ensuring that you don't enter a duplicate record into the table. For example, in a Customer table each customer would have a unique identification number. To set a Primary Key, click the View button to switch to Design view.
Step 2: Select the Field

Click in the Field Name column of the field you want to edit. Notice that the current field indicator displays to the left of the current field name, as shown in this screen.
Step 3: Set the Primary Key

Choose Edit and select Primary Key, or you can right-click the row and choose Primary Key from the shortcut menu. As a result, a key symbol appears in the row selector for the primary key field. If no single field uniquely identifies each record in the table, you can use multiple fields. To select more than one field as the primary key, hold down the Ctrl key and click the row selector for each additional field.
Step 4: Switch to Datasheet View

From Design view click the View button. The button appears with a drop-down arrow to display the text name for the button as well as the icon. You are prompted by Access to save the table design. If you choose Yes, the changes you made will be saved and displayed in Datasheet view. If you choose No, the changes you made will not be saved, and the design in Datasheet view will not be changed.
