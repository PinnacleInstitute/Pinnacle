Make the Transition to Access 10
Step 1: New Features

Excel now supports Extensible Markup Language (XML). Users can load and save generic, well formed, and spreadsheet XML directly into Excel. Furthermore users can query XML spreadsheets from the Web and thereby analyze important Web data.
Step 2: Office Web Components

A variety of improvements have been made to the Excel Web Components, such as support for loading native Excel files, support for named ranges, multiple worksheets, wrapped texts, and the ability to publish entire workbooks with interactivity to the Web. The Excel PivotTable® component offers conditional filtering on a range of expressions for filtering data. Finally, Office XP extends the object model for the components that allow developers to build custom solutions that deliver data to the browser.
Step 3: Smart Tags

Microsoft Office Smart Tags are a set of buttons that are shared across the Office applications. These buttons appear when the user needs them (such as when a user makes an error in an Excel formula, when Word automatically corrects a user’s action, or when a user pastes some data) and gives the user the options they need to change the given action or error.
Step 4: Paste Options

Multiple paste options allow users to decide whether they want to paste their data as they originally copied it, change the style so that it fits the style of the document they are pasting the data into, or apply specific characteristics to the data based on the content.

Main Elements of Access
Step 1: The Database Window

In Access, all objects of a database are stored in a single file, and the filename has an .MDB extension. You manage objects through the Database window. At the left of the Database window are tabs representing each of the six main object types: Tables, Queries, Forms, Reports, Macros and Modules. Selecting a tab switches the view of the window to reveal a listing of the current objects under that type.
Step 2: Tables

Tables are the primary building blocks of any Access database. All data is stored in tables. Every table in your database should focus on one subject, such as Customers, Orders, or Projects. The characteristics of each customer are separated into the fields. For example, First Name, Last Name, etc.
Step 3: Queries

A query is a question that you ask of the data stored in the tables of your database. For example, you can create a query that asks for only those customers in the state of Washington or that finds all employees with birthdays in the current month.
Step 4: Forms

Access forms serve two functions. The first is to present the table or query’s data in a format that is easy to view or update. These fields of data are available to use on a form when it is bound to a table or query. You can then edit the form just as you would edit the datasheet of the bound table or query, you can set parameters that restrict the form’s use to only viewing data, only adding data, or only editing data without the ability to add records.

Use the Database Wizard
Step 1: Choose a Database

From the New dialog box, look through the wizards on the Databases tab to find the type of database you want to create, and then double-click the wizard name. To create an address book database, for example, double-click Contact Management.
Step 2: Name the Database

First, name your new database file. Either accept the name supplied by Access, or type your own in the File name text box. Make it something that will be easily identifiable the next time you want to use the database. Click Create to continue.
Step 3: Determine the Structure

The next dialog box tells you that you are creating a database, click Next to continue. The next box has options for the actual structure of your database, including the tables that will be created and the fields to be included. Choose the table from the left, select (or deselect) the fields to use in the right and click Next.
Step 4: Choose a Screen Display

The next dialog box offers you several choices of backgrounds for screen displays and forms in the database. Click each name in the list to display the sample background. Choose a background and click Next.
Step 5: Choose a Style

From the next dialog box, choose the style for your printed reports, and then click Next to continue.
Step 6: Select a title for your database

Select a title for your database (it can be the same, or different from the filename used in step 2). Indicate whether to include a picture on all reports. (If you choose to include a picture, you will have to locate the picture file you want to use by clicking the Picture button). Click Next to continue.
Step 7: Click Finish

In the final dialog box, you can choose to start the database immediately. Click Finish to create your new database.

Navigate the Access Window
Step 1: View the Access Window

The Access program window consists of a title bar, menu bar, Database toolbar, and status bar. Each of these elements works the same as they do in the other Office programs.
Step 2: Use the Switchboard Window

When you create a database using the Database Wizard, the Main Switchboard window opens and the Database window is minimized onscreen. You can use the Switchboard to begin entering data into your database. To close the Switchboard window at any time, click its Close (X) button.
Step 3: Resize the Database Window

As you're using different parts of your database, the Database window is often minimized onscreen. To restore the window to its default size, click the Restore button. To maximize the window, click its Maximize button. (To minimize the window again, click its Minimize button). To close the database file completely, click the Close button.
Step 4: Use the Database Window

When the Database window is maximized or shown at its default size, you will notice it has several tabs that organize the elements of your database file. Click the tab to see its contents. To open any item in the Database window, double-click the item's name.
Step 5: The Database Buttons

The Database window has three buttons. Click the Open button to open the selected item (or double-click the item's name). Click the Design button to open the selected item in Design view. Click the New button to start a new item in your database.

Database Options
Step 1: Open the Options Window

You can change the way Access works by changing settings options. These options will stay in effect for all databases you open. To open the Options window, choose Tools and select Options. The View tab displays.
Step 2: Change View Options

If you want to change the way Access appears on the screen, you can change options on the View tab of the Options dialog box. For example, if you want to keep the Status bar from displaying on the screen, click the option to remove the check mark.
Step 3: Change General Options

You can change general options on the General tab of the dialog box. For example, you can change the default database drive and folder, choose to list the last 4 files opened on the bottom of the File menu, or change the default print margins.
Step 4: Change Edit/Find Options

The Edit/Find tab of the dialog box allows you to change the way the Find option works. You can also save time by not displaying the Confirm dialog boxes when you perform such actions as deletions.
Step 5: Change Keyboard Options

Using the Keyboard tab, you can change how the Enter key works when you add records to a table in the Move after Enter section of the dialog box. You can also change how the arrow keys on the keyboard work. In the Behavior entering field section, you can change what happens when the insertion point enters a field.
Step 6: Change Datasheet Options

On the Datasheet tab, you can change the default colors used to display the datasheet, the default font used for text, the default column width, and whether gridlines display. After making selections on any tabs of the dialog box, click the Apply button. When you have made all the desired changes, click the OK button to close the dialog box.

Design a Database
Step 1: Determine all Fields

Gather and analyze all the data, such as forms and reports, which the database application will use. For example, if you need to compile statistics about female employees, you must have a field in a table in the personnel database that contains that information. List all the data elements you will need to break each one down. For example, instead of a Name field, divide the data into title, first name, middle initial, last name, etc., making each element a separate field.
Step 2: Group Related Fields

Organize the fields by their subject content. For example, all information about a customer would be grouped together, and all information about salespeople would be grouped together. These groups of fields will become the tables in your database.
Step 3: Create Tables

Once you have related fields grouped together, create and name the tables that will contain the fields. Being organized before you start working with Access will save time in the long run. This will keep you from having to redesign your database later due to design flaws.
Step 4: Normalize the Data

When you normalize the data, you eliminate redundancy from your database. For example, if you reenter a customer's contact information each time they place an order, you will have duplicate data which leads to errors, wastes time, and takes up storage space. The solution is to break the data that contains the customer information and order information into separate tables then create a relationship between the two tables.

Backup a Database
Step 1: Exit from Access

You will use Windows Explorer to make a duplicate copy of an entire database file. Save any open tables; choose File, and then Exit. You will be returned to the Windows display.
Step 2: Open Windows Explorer

To open Windows Explorer, click the Start button on the status bar, choose Programs, and select Windows Explorer. Select the folder that contains the database you want to copy and if necessary scroll down and select the desired file.
Step 3: Copy the File

Choose Edit and select Copy or click the Copy button on the Windows Explorer toolbar. The file has been copied to the Clipboard.
Step 4: Paste the File

Click the drive and or directory where you want to copy the file. When you have opened the desired location, choose Edit and select Paste or click the Paste button on the Windows Explorer toolbar. The file is copied to the location that you designated. Close the Windows Explorer window by clicking the Close button.

Create Tables in Design View
Step 1: Open Design View

Use Design View to create or modify the design of a table, but not its record content. A table design contains the table's fields and their properties. From the Database window, double-click the Create Table in Design View option.
Step 2: Enter Field Names

The Table window opens in Design view, as shown in this figure. Type the Field Names for the table in the first column. Each name must be unique, that is, each field must have a different name. A field name can be a maximum of sixty-four characters in length. Don't begin a field name with a space. Although you can include spaces within the field name, it is best to avoid this as it can cause problems with complex applications. Instead of Customer Number, type CustomerNumber or Customer_Number.
Step 3: Enter a Description

You can enter a description for each field in the Description column. This is useful for documenting the field so that others may understand its use, so it is a good habit to include a brief description.
Step 4: Choose a Data Type

The Data Type determines what kind of data a field can store. Fields containing data such as names, addresses, or phone numbers use the Text data type. An all-numeric field that might be used in a calculation, such as quantity or amount, uses the Number type. Dollar amounts use Currency. The AutoNumber type numbers each record sequentially. Click in the Field Type text box, and click the drop-down arrow to display data type options shown in this figure.
Step 5: Save the Table Design

Save the table by clicking the Save button or choose File, Save As. The Save As dialog box opens where you type the table name. Access then prompts you to give the table a primary key. If you select Yes, Access adds a key field to your table. Selecting No avoids setting a primary key, and you can specify it at a later time using Design view.

Navigate Tables
Step 1: View the Current Record

Access tables display the data that has been entered. The data is divided into elements called fields, and all the fields associated with one entry in the table make up one record. When you are viewing a table, one record is the current record. The current record symbol appears to the left of the first field of the current record. Notice that Table 1 is the current record in this figure. When you click on another record, the symbol moves to that record, making that record the current record.
Step 2: Use the Scroll Bars

You will often have more records in a table than you can view on the screen. To view records below the current display, use the vertical scroll bar on the right edge of the window. You may also have more fields in the table than you can view on the screen. To view fields to the right of the current display, use the horizontal scroll bar along the bottom edge of the window.
Step 3: Use the Navigation Buttons

If you need to make a record of the current record and you can't see it on the screen, the easiest method is to use the navigation buttons located at the bottom of the table window. Notice that the navigation toolbar keeps track of the current record for you. Click the single arrows to move to the next or back to the previous record. Click the arrows with the lines to move to the first record in the table or to the last record in the table.
Step 4: Add a Blank Record

To create a blank record at the end of the table, click the button with the asterisk. The insertion point moves to the blank row at the bottom of the table, allowing you to type a new record.

Rename a Table
Step 1: Select a Table

Occasionally, you might need to change a table name. For example, if you want to revert to a backup copy of the table, you would want to rename the backup copy to the original table name. To select a table to rename, click the Table button on the Objects bar in the Database window. Select the appropriate table
Step 2: Type a New Name

With the table selected, choose Edit and select Rename. The table name appears in edit mode, allowing you to type the new name. Press Enter to save the new name, and the table appears in the list.
Step 3: Use an Existing Name

If the name you type for the table in edit mode is the name of an existing table, Access displays a warning that the original table will be overwritten, and all the data will be replaced with the current table. If you want to replace an existing table by overwriting it, then choose Yes, otherwise, click No. If you choose the No option, you will need to choose another name for the table.
Step 4: Undo the Operation

If you rename the table from the database, you can immediately click the Undo button on the toolbar to restore the previous table name. However, if you have performed any other actions, you will not be able to undo the operation.

Delete a Table
Step 1: Select the Table

Occasionally, you might have a table that needs to be deleted, such as an out-of-date backup table. To select a table to delete, click the Table button on the Objects bar in the Database window and select the appropriate table.
Step 2: Delete a Table

With the table selected, choose Edit and select Delete or press the Delete key. Access displays a warning asking you if you want to delete the table. If you click the No button, the table will not be deleted, but if you click the Yes option, the table will be deleted from the database.
Step 3: Undo the Deletion

If you delete the table from the database, you can immediately click the Undo button on the toolbar to restore the table. However, if you have performed any other actions, you will not be able to restore the table.
Step 4: Deleting Related Tables

If you select a table to delete that is related to another table in the database, Access displays a warning message that the table is related to another table and can't be deleted until the relationship is deleted. If you want Access to delete the relationship, click the Yes option in the dialog box. If you decide not to delete the relationship, click the No option.

The Table Wizard
Step 1: Start the Table Wizard

The Table Wizard is a handy feature that designs your tables for you. From the Database window, click the Tables object in the Objects bar, and then double-click the Create Table by Using the Wizard option. This opens the Table Wizard, as shown in this figure. You can view two separate table lists, Business or Personal. Select the one you want.
Step 2: Add Fields

You can build your table using fields from different sample tables, just mix and match what you need. Select a table from the Sample Tables list, and the Sample Fields list box. To add a field, select the field and click the > button. To remove fields from the Fields in my new Table list box, select the field and click the < button. Select a field and click the Rename Field button to type a new name for a field. When you have finished adding fields, click the Next button.
Step 3: Name the Table

In the next dialog box, enter a descriptive name for the table. The dialog box asks you if you want the wizard to set the primary key. If you choose Yes, set a primary key for me, Access will determine the field that will uniquely identify each record in the table. If you choose No, I'll set the primary key, you can determine for yourself later what field you want as the primary key. When you are finished, click the Next button.
Step 4: Finish the Table

If your database already has at least one table, the next dialog box that appears asks about the relationship between tables. Click Next for now to open the final dialog box. Indicate whether you want to change the table design, enter data directly or have the wizard create a data entry form for you and then click Finish.
